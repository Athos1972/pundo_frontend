# Internationalisierung (I18n)

## Unterstützte Sprachen

| Code | Sprache | RTL |
|---|---|---|
| `en` | Englisch | Nein |
| `de` | Deutsch | Nein |
| `el` | Griechisch | Nein |
| `ru` | Russisch | Nein |
| `ar` | Arabisch | **Ja** |
| `he` | Hebräisch | **Ja** |

## Sprachauswahl

- Cookie: `pundo_lang` (1 Jahr, SameSite=Lax)
- Server: `getLangServer()` aus `src/lib/lang.ts` — liest Cookie via `next/headers`
- Client: `getLangFromCookie()` aus `src/lib/lang.ts`
- Fallback: `'en'`

## RTL-Behandlung

- `isRTL(lang)` → `true` für `ar`, `he`
- Root Layout setzt `<html lang={lang} dir={rtl ? 'rtl' : 'ltr'}>`
- Tailwind: `rtl:` Modifier für spiegelbare Layouts (`rtl:text-right`, `rtl:flex-row-reverse`)
- **Niemals im Frontend raten** — `rtl`-Flag immer aus API-Response oder `src/lib/lang.ts` lesen

## UI-Strings hinzufügen

1. Neuen Key zur englischen Locale in `src/lib/translations.ts` hinzufügen
2. Für alle anderen 5 Sprachen übersetzen (gleiche Datei)
3. `Translations`-Typ wird automatisch inferiert — TypeScript wirft Fehler wenn ein Key fehlt

```typescript
// src/lib/translations.ts
export const translations = {
  en: {
    my_new_key: 'My new string',
    // ...
  },
  de: {
    my_new_key: 'Mein neuer Text',
    // ...
  },
  // ... alle 6 Sprachen
}
```

## Dynamische Inhalte

Produktnamen, Kategorienamen und Shop-Adressen kommen übersetzt vom Backend — kein Frontend-Mapping nötig.  
Das Backend liest den `Accept-Language`-Header und liefert die passende Übersetzung.

## Translation-Key-Referenz

Alle Keys: `src/lib/translations.ts` → `typeof translations.en`

Wichtige Keys:

| Key | Typ | Verwendung |
|---|---|---|
| `price_on_request` | `string` | Label für Angebote ohne Fixpreis |
| `price_free` | `string` | Label für kostenlose Angebote |
| `price_variable` | `string` | Label für variable Preise |
| `filter_price_only` | `string` | Filter-Chip auf Suche/Produktseite |
| `contact_shop` | `string` | CTA-Button bei `on_request`-Angeboten |
| `available` | `string` | Verfügbarkeits-Badge |
| `unavailable` | `string` | Nicht-verfügbar-Badge |
| `days_ago` | `(n: number) => string` | Relative Zeitangabe |
| `subcategories` | `(n: number) => string` | Kategorie-Zähler |
